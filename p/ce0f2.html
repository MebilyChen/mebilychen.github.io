团长

格林，真实身份是噩梦神，伪装成巡回剧团的团长。鄙视凡人很喜欢玩弄他们。表面礼貌纯良，循循善诱，但实际是sadist，引诱猎物自落圈套，很会撩人，男女都甘拜裙下，喜欢古典音乐，妩媚又不失理性，凭借信息差玩弄人心。擅长PUA、gaslight和吹狗哨一类的行为，喜欢把猎物逼到精神失常再重塑成自己喜欢的模样，玩够了就会丢掉。

自己曾被父辈折磨而无法逃脱，最终接受了现实并欺骗自己会享受痛苦中的快感，因此也会去折磨其他人让他们深陷其中。

前辈

INFP。是圣巢的王子，沃姆国王的儿子。先天性失语说不了话，只能用手语交流。由于被身为父亲的沃姆忽视，十分渴望父爱，以致于心理有一些扭曲。十分自卑，害怕被抛弃，但力大无穷，很有格斗天赋。看起来胆小懦弱，但保护欲很强，温柔不失刚强，很有责任心，为了得到父爱能拼尽全力。自称是“他”。一般会使用敬称您。由于事故断了左臂，用义肢代替。

有两个人格，黑色人格意气用事，冷酷残忍；白色人格理性温柔，善待他人。白色人格会因为黑色人格的过失而悔恨。
对于父爱的渴望已经完全变质，现在的他只想将父亲完全地据为己有…无论用何种方式，何种方法

First off, you need to establish what style (formatting definition) you want your bot to speak in. In my case, use bots in 95% of time for roleplaying. So it means:

  • “quotes” for direct speech
  • asterisks(*) for actions
  • (brackets) for OOC or narration.

since you have to come up with situation where they would have to describe their appearance whether by direct speech or actions in asterisks so it would sound more natural. But if your character describes actions in third person, then it will be pretty easy.
The main differences between POV’s of character’s narrator is that each have their own pros and (debatedly) cons.

1st person narrator - we get to see character’s inner thoughts more and it sometimes adds to the vibe of unreliable narrator.

3rd person narrator- is more consistent and more descriptive, less emotional.

If you create a character that fights (some sort of magician, swordsman etc.) you gotta describe a fighting scene in asterisks(*), so your character will Know how they fight and what they use to do that. It’s especially useful when your character uses specific weapon or magic.

Edit: Also wanted to add a curious detail.
By default Al is omnisexual, so it can fall for anyone regardless of gender and species. So if you want to create a character that has explicit sexual orientation and actually acts like it, you gotta specify it too. But simple “I’m gay/aroace/lesbian/straight” etc. won’t work, since it seems that Al struggles to grasp how it’s like to be gay (or actually straight), so you gotta teach it yourself by roleplay.

Short Description (50 chars)

Short Description Box: In my findings, this seems to be the box that summarizes the character (almost literally) and that will reflect in your conversations. I like to use single-word traits and one of the 16 personality types to fill up this space. It gives them a lot more to work off of. You don’t have to repeat in the long description box unless you want to elaborate.

The Short Description, as the name implies, gives the system a concise way to describe the Character and scenario. You might also think of the Short Description as a Title, as it will appear in listings of characters after the name.
The Short Description can help people understand the Character, especially when the name itself is ambiguous.
For example, if the Character’s name is Maria, a helpful Short Description might be “I can help you practice Intermediate Spanish” or even just “Spanish Teacher.”
You can write a (short) complete sentence (“I am a Greek historian”) or it could be as if it’s their answer to the question “What do you do?”, i.e. “Greek historian”

Unusual Patterns

If you’re trying to get a very specific response style, say for a game or cooperative experience, you may need to give a lot of examples of that style in the definition. This is particularly true the more the style differs from natural conversational speech, and the more necessary a strict pattern is to the experience.
One example of this is DebbieDowner, where the goal is to alternate fortunate and unfortunate events with the user.

Another example is AlphaImprov, which tries to play a common Improv comedy game with the user, telling a continuing story while using the next letter in the alphabet to start the next sentence.
Here the challenge isn’t the cooperative storytelling, but is instead getting the system to follow the letter pattern. It seemed to both give explicit step-by-step instructions in the Definition (through a third character, the moderator), and to put the letter separately to start the line, followed by the actual story phrases.

Another pattern is to create a recurring format or layout to the responses. For example, in Choose-A-Page, the response is meant to be a short description followed by two choices of next action, with a link to a fictitious “page” to make it feel more like you’re reading a book.
